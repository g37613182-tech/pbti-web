# PBTI 趣味互动测试网页开发计划

## 1. 摘要 (Summary)
本项目旨在开发一个具有“搞怪/抽象梗风”的 PBTI 性格测试互动网页。该网页的核心目标是为用户在“醒图”App上创作的 PBTI/SBTI 相关特效玩法进行宣传和导流。网页将包含搞怪的首页引导、经典的翻页单选题测试，以及带有强烈“去醒图App体验”引导的结果展示页。

## 2. 当前状态分析 (Current State Analysis)
当前项目目录为空（全新的项目）。我们需要从零开始搭建前端工程。

## 3. 核心玩法与内容构思 (Core Gameplay & Content)
为了契合“搞怪/抽象梗风”，整体文案和视觉将采用贴近年轻人的网络热梗风格。
*   **首页 (Home)**：
    *   **视觉**：夸张、高饱和度的配色或极具反差感的抽象排版。
    *   **文案**：主标题大字展示“PBTI 测试 - 测测你的隐性抽象人格”，副标题明确告知“更多专属搞怪特效，请去醒图体验”。
    *   **交互**：点击“开始发疯/测测看”按钮进入答题。
*   **测试页 (Quiz)**：
    *   **交互**：采用**经典翻页单选题**形式，共设置 4-5 道题目。点击选项后自动平滑过渡（翻页）到下一题。
    *   **内容**：题目设定为荒诞、搞笑的日常场景（例如：“当老板周末半夜在群里@你，你的第一反应是？”），选项对应不同的 PBTI 维度（模仿 MBTI 的四个维度，但字母可设定为 P/S, B/N, T/F, I/E 等，最终组合出如 PBTI, SBTI 等结果）。
*   **结果页 (Result)**：
    *   **内容**：根据用户的选项积分，计算出其“抽象人格”类型（如：SBTI - 纯爱战神，PBTI - 顶级牛马等），并附带一段搞怪的判词。
    *   **核心转化**：在结果页最显眼的位置，放置一个跳动或极具视觉冲击力的按钮：“👉 去醒图 App 解锁你的专属 PBTI 特效 👈”，点击后可引导用户去往醒图（可配置为跳转链接或展示二维码）。

## 4. 技术栈与架构 (Tech Stack & Architecture)
考虑到网页需要良好的交互动效以及主要在移动端传播，推荐使用以下现代前端技术栈：
*   **框架**: React (使用 Vite 构建，轻量且极速)
*   **样式**: Tailwind CSS (便于快速编写响应式、移动端优先的 UI 样式)
*   **动效**: Framer Motion (用于实现页面切换、选项点击的丝滑翻页动画)
*   **部署**: 纯静态前端网页，后续可轻松部署至 Vercel, GitHub Pages 或任意静态服务器。

## 5. 提议的更改 / 实现步骤 (Implementation Steps)
1.  **项目初始化**：在当前目录下通过 `npm create vite@latest . -- --template react` 初始化项目，并安装 Tailwind CSS 与 Framer Motion。
2.  **目录结构划分**：创建 `src/components`（存放 Home, Quiz, Result 组件）、`src/data`（存放搞怪题库和结果映射配置）、`src/utils`（存放计分逻辑）。
3.  **开发主页 (Home)**：实现夸张的标题UI和醒图App的初步口播，添加进入测试的按钮。
4.  **开发测试页 (Quiz)**：实现题目的遍历渲染，加入选项点击事件和 Framer Motion 的左右滑动/淡入淡出翻页动效。
5.  **开发结果页 (Result)**：实现逻辑分数统计并匹配对应的结果文案，重点设计“去醒图App”的强引导按钮。
6.  **移动端适配与UI打磨**：确保网页在手机屏幕下比例完美，按钮易于点击，字体清晰。

## 6. 假设与决策 (Assumptions & Decisions)
*   **Mobile-First 优先**：假设该网页主要通过微信群、朋友圈等移动端社交平台传播，因此 UI 设计将完全基于移动端尺寸（窄屏）进行优化。
*   **纯前端计算**：为了保证轻量和加载速度，不引入后端和数据库。所有题目数据和计分逻辑硬编码在前端，用户的测试结果仅在本地浏览器中计算展示。

## 7. 验证步骤 (Verification Steps)
1.  运行 `npm run dev` 启动本地服务器。
2.  在浏览器中打开，并使用开发者工具切换至手机预览模式。
3.  走通完整的测试流程（首页展示 -> 顺畅答完 5 道题 -> 查看最终结果页）。
4.  确认结果页的“去醒图App”按钮是否醒目，以及整体风格是否符合“搞怪/抽象梗风”。
